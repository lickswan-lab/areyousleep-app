# 床前 App — 真实用户测试方案

## 一、测试目标

验证 Phase 2 功能是否达到「小红书上线」标准：
1. 核心流程无阻塞（首次打开 → 写担忧 → 封存 → 呼吸引导 → 完成）
2. 新功能体验流畅（情绪引导、睡眠日记、脑科学内容、周报卡片）
3. 文案打动人（不生硬、不说教、像朋友）
4. 视觉在手机截图上好看（小红书传播核心）

## 二、招募标准

### 目标用户
- **数量**: 8-12 人
- **年龄**: 20-32 岁
- **核心筛选条件**: 最近1个月有3次以上"躺下后30分钟睡不着"经历
- **排除**: 正在接受睡眠障碍治疗的用户

### 三类画像各招 3-4 人
| 画像 | 描述 | 招募渠道建议 |
|------|------|------------|
| A 职场型 | 22-28岁，加班后脑子停不下来 | 即刻、小红书「打工人」话题 |
| B 学生型 | 18-24岁，考试/考研焦虑 | 小红书「考研」「期末」话题 |
| C 情绪型 | 24-32岁，情绪敏感、容易反刍 | 小红书「情绪管理」「高敏感」话题 |

## 三、测试流程（每人约 25-30 分钟）

### Step 1: 暖场（3分钟）
```
"谢谢你参加测试！这不是考试，没有对错。
你的所有感受和吐槽都是有价值的。
我会让你试用一个帮助睡眠的小工具，然后聊聊感受。"
```

**破冰问题**:
- "最近睡眠怎么样？有没有躺下之后脑子停不下来的情况？"
- "一般是什么事情让你睡不着？"

> 记录用户自我描述的画像类型（A/B/C）

### Step 2: 自由探索（8分钟）
```
"现在请你打开这个App，就像你真的在睡前使用一样。
想做什么就做什么，不用问我。
如果你有什么想法，随时说出来。"
```

**发送链接**: https://chuangqian.vercel.app

**观察记录**（不干预用户）:

| 观察维度 | 记录内容 |
|---------|---------|
| 首屏反应 | 用户看到首页的第一句话/表情 |
| 状态选择 | 选了哪个（工作/情绪/说不清）？犹豫了吗？ |
| 写担忧 | 写了几条？写了什么类型？思考时间多久？ |
| AI 回复 | 看到AI回复时的反应？读了吗？ |
| 封存行为 | 是逐条封存还是选择批量？ |
| 呼吸引导 | 选了7分钟还是3分钟？做完了吗？中途退出了吗？ |
| 底部入口 | 是否注意到「为什么睡不着」「周报」「睡眠日记」？ |
| 情绪引导 | （如果选了情绪）选了哪些情绪标签？全程参与了吗？ |

### Step 3: 定向任务（8分钟）

如果用户自由探索时没有触达以下功能，引导体验：

**任务 1 — 脑科学内容**
```
"你有没有注意到底部有个'为什么睡不着'？点进去看看，
随便选一篇读读，告诉我感觉怎么样。"
```

**任务 2 — 睡眠日记**
```
"试一下'睡眠日记'功能，假装记录一下昨晚的睡眠。"
```

**任务 3 — 周报卡片**
```
"点开'本周报告'看看。如果你的朋友在小红书刷到这个卡片，
你觉得她会想试试这个App吗？"
```

**任务 4 — 情绪引导（仅 Persona C 用户）**
```
"回到首页，选'情绪、人际、心里的事'，体验一下里面的引导。"
```

### Step 4: 深度访谈（10分钟）

#### 核心体验
1. "用一句话描述这个App在帮你做什么？"
2. "整个过程中，有没有哪个瞬间让你觉得'哦，这个不一样'？"
3. "有没有哪个地方让你觉得别扭或者不舒服？"

#### 功能反馈
4. "写担忧这个环节，你觉得有用吗？为什么？"
5. "AI 给你的回复，你怎么看？（太假？太短？还是刚好？）"
6. "呼吸引导你选了X分钟版，为什么？做完的感觉怎么样？"
7. "脑科学那几篇文章，看了之后有什么感觉？"

#### 分享意愿
8. "如果你睡前真的用了这个，第二天你会发小红书吗？为什么？"
9. "周报卡片的设计，你觉得发到小红书好看吗？缺什么？"
10. "你觉得这个App值多少钱？免费的话你会坚持用吗？"

#### 留存预判
11. "你觉得你会连续用几天？什么会让你不想用了？"
12. "如果只能保留一个功能，你选哪个？"

### Step 5: 收尾（2分钟）
```
"最后一个问题：如果你要给一个朋友推荐这个App，
你会怎么用一句话介绍它？"
```

> 记录用户的原话 — 这是最佳营销文案来源

## 四、量化评估表

每位测试用户填写（测试结束后）:

| 维度 | 评分（1-5） | 说明 |
|------|-----------|------|
| 第一印象 | __ | 打开App的第一感觉 |
| 写担忧体验 | __ | 写下来之后的感觉 |
| AI 回复质量 | __ | 回复是否让你感到被理解 |
| 封存仪式感 | __ | "放进盒子"的动画和感觉 |
| 呼吸引导 | __ | 跟着做完的感觉 |
| 情绪引导 | __ | 情绪命名和容器的感觉 |
| 脑科学内容 | __ | 看完后对失眠的理解 |
| 整体帮助 | __ | 你觉得这能帮你睡着吗 |
| 推荐意愿 | __ | 你会推荐给朋友吗 |
| 付费意愿 | __ | 你愿意为此付费吗 |

**NPS 问题**: "0-10分，你有多大可能向朋友推荐这个App？"

## 五、关键指标 & 达标标准

| 指标 | 小红书上线标准 | 测量方式 |
|------|-------------|---------|
| 任务完成率 | ≥ 90% 完成核心流程 | 观察记录 |
| 首屏理解度 | ≥ 80% 用户5秒内理解App定位 | 暖场问答 |
| AI 回复满意度 | 均分 ≥ 3.5/5 | 评估表 |
| NPS | ≥ 30 | NPS计算 |
| 分享意愿 | ≥ 50% 表示"会发小红书" | 访谈问题8 |
| 周报卡片美观度 | ≥ 70% 认为"好看/想分享" | 访谈问题9 |
| 留存预期 | 均值 ≥ 5天 | 访谈问题11 |
| 核心差异化 | ≥ 60% 说出"和其他助眠App不同" | 访谈问题2 |

## 六、测试后行动

### 若达标
1. 小红书内容准备（截图 + 使用场景文案）
2. 测试用户转化为种子用户，邀请写真实使用笔记
3. APK 更新为 Phase 2 版本

### 若不达标
按严重程度排序修复：
- **P0（阻塞上线）**: 用户无法完成核心流程的bug
- **P1（体验问题）**: NPS < 30 的对应环节
- **P2（优化项）**: 文案调整、动画打磨

## 七、小红书冷启动内容模板

### 笔记类型 1: 个人体验
```
标题: "找到了一个让我放下手机就能睡着的App"
内容:
- 痛点共鸣（睡前脑子停不下来）
- 使用过程（写担忧→放进盒子→呼吸引导）
- 核心感受（"不是让你放松，是帮你把脑子卸货"）
- 周报截图
```

### 笔记类型 2: 数据型
```
标题: "用了7天，我发现85%的担忧第二天都没发生"
内容:
- 脑科学解释（为什么越努力越睡不着）
- 个人周报数据截图
- 对比：用之前 vs 用之后
```

### 笔记类型 3: 安利型
```
标题: "姐妹们！终于有个助眠App不是放白噪音的了"
内容:
- 和传统助眠App对比
- 功能亮点截图
- "写完担忧那一刻真的松了口气"
```
